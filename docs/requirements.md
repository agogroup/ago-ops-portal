# 機能要件書

## ステータス: 設計中

---

## 1. プロジェクト背景

### 解決したい課題

1. **TEKO配車の非効率**
   - 工程表を見て「いつゴミが出るか」を人が判断している
   - 複数現場のルート最適化ができていない

2. **情報共有の遅延**
   - 現場状況が関係者にリアルタイムで伝わらない
   - 「今どの現場がどうなっているか」の把握に時間がかかる

3. **連絡先の散在**
   - 職人・業者の連絡先が個人のスマホに分散

### 期待する効果

- TEKO配車の判断時間短縮
- 現場状況の可視化
- 連絡先の一元管理

---

## 2. ユーザーストーリー

### 染谷雅人（TEKO現場管理）として

- [ ] 今日・明日のTEKO配車予定を一覧で見たい
- [ ] 各現場のゴミ発生タイミングを把握したい
- [ ] 配車ルートを効率的に組みたい

### 菅優伍（TEKOデスク管理）として

- [ ] 全現場の進捗を一目で確認したい
- [ ] 染谷さんへの報告資料を簡単に作りたい
- [ ] 学習のため、過去の配車履歴を振り返りたい

### 堀川寿人/トラ（現場管理）として

- [ ] 自分が担当する現場の工程を確認したい
- [ ] TEKOの到着予定時刻を知りたい
- [ ] 職人への連絡先をすぐ調べたい

### 梅内光（積算・工程準備）として

- [ ] 作成した工程表を関係者に共有したい
- [ ] 工程変更時に関係者へ通知したい

## 3. 機能一覧（優先度順）

### Phase 1: 最小限の価値提供

| 機能 | 概要 | 優先度 |
|------|------|--------|
| 現場一覧表示 | 進行中の現場を一覧表示 | 高 |
| TEKO配車カレンダー | いつどの現場に行くか表示 | 高 |
| 職人連絡先検索 | 名前で検索、ワンタップ発信 | 高 |

### Phase 2: 効率化

| 機能 | 概要 | 優先度 |
|------|------|--------|
| ゴミ発生タイミング自動算出 | 工程から「この日にゴミが出る」を表示 | 中 |
| 配車ルート提案 | 複数現場を効率的に回る順序を提案 | 中 |
| 工程変更通知 | 変更時にLINE通知 | 中 |

### Phase 3: 高度化

| 機能 | 概要 | 優先度 |
|------|------|--------|
| 配車履歴分析 | 過去データから傾向を分析 | 低 |
| 産廃コスト試算 | ゴミ量から処理費用を概算 | 低 |

## 4. 非機能要件

### 利用環境

- スマートフォン（主）
- PC（副）

### 性能

- 一覧表示: 2秒以内
- 検索: 1秒以内

### セキュリティ

- 関係者のみアクセス可能
- 機密情報は含めない設計
