# 画面モックアップ

## 概要

スマートフォン表示を主として設計。
タイムチャートがメイン画面。

---

### 色分け（案）

| 色 | 意味 |
|----|------|
| 青 | TEKO（ゴミ回収） |
| 緑 | 現場管理（トラ等） |
| オレンジ | 内装工事 |
| 赤 | 解体 |
| 黄 | 電気工事 |
| グレー | 空き（予定なし） |

## 7. ナビゲーション構造

```
[タイムチャート] ←→ [現場一覧] ←→ [スケジュール登録] ←→ [連絡先]
      │                 │
      │                 └── [現場詳細]
      │
      └── セルタップ → [スケジュール編集]
```

### 下部ナビゲーション

| アイコン | 画面 | 説明 |
|----------|------|------|
| 📊 | タイムチャート | メイン。誰がどこにいるか |
| 🏢 | 現場一覧 | 現場の進捗確認 |
| ➕ | スケジュール登録 | 予定の追加 |
| 📞 | 連絡先 | 電話帳 |

---

## 8. スマホ最適化のポイント

1. **片手操作**: 主要操作は画面下部に配置
2. **大きなタップ領域**: ボタン・セルは指で押しやすいサイズ
3. **横スクロール最小化**: タイムチャートは縦スクロール中心
4. **ワンタップ発信**: 連絡先から直接電話
5. **オフライン対応**: 直近データはキャッシュ（Phase 2）
