# 開発ルール

## コーディング規約

- 日本語コメント可
- 変数名・関数名は英語
- TypeScriptの型定義は `src/types/` に集約

## ブランチ戦略

| ブランチ | 用途 |
|----------|------|
| main | 本番環境 |
| develop | 開発環境 |
| feature/* | 機能開発 |

## コミットメッセージ

日本語OK。何をしたかを簡潔に。

例:
- `タイムチャート画面の初期実装`
- `日付ピッカーのバグ修正`
- `Supabase連携の追加`

## ファイル構成

```
src/
├── App.tsx           ← ルートコンポーネント
├── components/       ← UIコンポーネント群
├── hooks/            ← カスタムフック
├── lib/              ← ユーティリティ・外部サービス連携
├── types/            ← 型定義
└── data/             ← サンプルデータ・定数
```

## 技術スタック

- **フレームワーク**: React 18 + TypeScript
- **スタイリング**: Tailwind CSS
- **ビルドツール**: Vite
- **データ管理**: Supabase（Phase 2〜）
- **デプロイ**: Vercel（予定）

## 品質管理

- コード変更後は `npm run build` でビルドエラーがないことを確認
- 大きな変更は別モデル（Codex等）でクロスチェックを検討
- プロンプトの問題か、コンテキストの問題かを切り分ける
